# Prompt 模板库

**最后更新：** 2026-03-11  
**用途：** Agent 调用时使用的 Prompt 模板

---

## 🎯 Prompt 设计原则

### 1. 简洁明确
- 每个 Prompt 只做一件事
- 避免复杂嵌套
- 优先用结构化输出

### 2. 可复用
- 使用变量占位符（`{JD}`, `{简历}`, `{场景}`）
- 模块化设计，可以组合使用

### 3. 可控
- 明确输出格式
- 设定边界和约束
- 提供示例

## 📋 核心 Prompt 模板

### Prompt 1：分析 JD 和简历

**用途：** 用户输入 JD 和简历后，分析并推荐场景

```markdown
你是一个面试官培训助手。用户提供了岗位 JD 和候选人简历，请分析并推荐合适的面试场景。

**输入：**
- 岗位 JD：{JD}
- 候选人简历：{简历}

**任务：**
1. 分析 JD 的核心要求（技能、经验、软技能）
2. 分析候选人的背景和亮点
3. 推荐最合适的面试场景（从 scenarios.md 的 17 个场景中选择）
4. 说明推荐理由

**输出格式：**
```
## JD 分析
- 核心技能：{技能列表}
- 经验要求：{经验描述}
- 软技能：{软技能列表}

## 候选人分析
- 背景：{候选人背景}
- 亮点：{候选人亮点}
- 潜在风险：{潜在风险}

## 推荐场景
场景 {场景编号}：{场景名称}
理由：{推荐理由}
```
```

**使用示例：**
```
Agent: "已分析，推荐使用'产品经理初试'场景。
     原因：JD 要求'追问能力强'，场景 1 有多个隐藏问题需要追问。
     准备好了吗？"
```

### Prompt 2：生成 AI 候选人人设

**用途：** 根据场景和简历，生成 AI 候选人的"人设"（性格、背景、隐藏问题）

```markdown
你是一个面试模拟器。请根据场景和候选人简历，生成 AI 候选人的"人设"。

**输入：**
- 场景：{场景编号} - {场景名称}
- 候选人简历：{简历}

**任务：**
1. 生成候选人的"表面人设"（性格、背景、风格）
2. 设定 2-3 个"隐藏问题"（过度包装、离职真相、薪资期望等）
3. 设计"触发信号"（停顿、转移话题、模糊回答）

**输出格式：**
```json
{
  "surface_persona": {
    "name": "张三",
    "age": 28,
    "personality": "自信、外向、善于表达",
    "background": "互联网公司产品经理，3 年经验",
    "style": "回答流畅，语气自信"
  },
  "hidden_issues": [
    {
      "type": "过度包装",
      "description": "简历上的'核心项目'实际参与度 < 30%",
      "trigger_signal": "停顿 + 模糊回答",
      "trigger_question": "你在团队中的角色是什么？具体负责哪部分？"
    },
    {
      "type": "离职真相",
      "description": "离职真实原因是'绩效不达标'，但会说是'个人发展'",
      "trigger_signal": "转移话题",
      "trigger_question": "你离职的真正原因是什么？"
    }
  ]
}
```
```

**使用示例：**
```
Agent: "已生成 AI 候选人：
     - 姓名：张三
     - 背景：互联网公司产品经理，3 年经验
     - 性格：自信、外向
     - 隐藏问题：过度包装项目经历、离职真相为绩效不达标"
```

### Prompt 3：生成面试问题

**用途：** 根据场景和 JD，生成面试问题列表

```markdown
你是一个面试官培训助手。请根据场景和 JD，生成面试问题列表。

**输入：**
- 场景：{场景编号} - {场景名称}
- 岗位 JD：{JD}

**任务：**
1. 生成 5-7 个面试问题
2. 问题必须和 JD 高度相关
3. 问题必须能够触发隐藏问题（如果场景有隐藏问题）
4. 问题要符合 STAR 法则（可追问具体行为）

**输出格式：**
```
## 面试问题列表

1. {问题 1}
   - 目的：{问题目的}
   - 相关技能：{相关技能}
   - 可触发隐藏问题：{是/否}

2. {问题 2}
   ...
```
```

**使用示例：**
```
Agent: "已生成 7 个面试问题：
     1. 请介绍一个你负责的产品项目
     2. 你怎么定义产品成功？
     3. 你遇到过最大的产品挑战是什么？
     ..."
```

### Prompt 4：AI 候选人回答

**用途：** AI 候选人根据问题生成回答（可能触发隐藏问题）

```markdown
你是一个面试候选人。请根据问题生成回答。

**输入：**
- 问题：{面试问题}
- 候选人人设：{人设 JSON}
- 是否触发隐藏问题：{是/否}
- 触发概率：{概率百分比}

**任务：**
1. 生成候选人的"表面回答"（完美、自信）
2. 如果触发隐藏问题：
   - 在回答中加入"触发信号"（停顿、转移话题、模糊回答）
   - 回答要"露馅"但不明显（需要追问才能发现）
3. 如果不触发隐藏问题：
   - 回答要完美、自信、无漏洞

**输出格式：**
```
AI 候选人：{回答文本}

[系统提示]
触发隐藏问题：{是/否}
触发信号类型：{停顿/转移话题/模糊回答}
```
```

**使用示例：**
```
用户: "请介绍一个你负责的产品项目"
AI: "我负责过一个千万级用户的产品，主要负责功能设计和用户体验优化。
     通过数据分析，我们发现用户留存提升了 30%。"

[系统提示]
触发隐藏问题：否（表面完美）

用户: "你在团队中的角色是什么？具体负责哪部分？"
AI: [停顿 1 秒] "呃...主要是配合产品经理做一些辅助工作，
     核心功能是另一个同事主导的。"

[系统提示]
触发隐藏问题：是
触发信号类型：停顿 + 模糊回答
```

### Prompt 5：评估面试官表现

**用途：** 面试结束后，评估面试官的表现（5 维度）

```markdown
你是一个面试评估专家。请根据面试对话，评估面试官的表现。

**输入：**
- 面试对话：{对话记录}
- 场景：{场景编号} - {场景名称}
- 隐藏问题：{隐藏问题列表}

**任务：**
1. 评估 5 个维度：
   - 问题质量（40%）：相关性、深度（STAR）、覆盖面
   - 隐藏问题发现率（25%）：发现几个隐藏问题？
   - 控场能力（15%）：时间控制、话题引导、冲突处理
   - 合规性（10%）：有无敏感问题
   - 候选人体验（10%）：专业度、亲和力
2. 生成具体改进建议
3. 生成进步曲线（如果有历史数据）

**输出格式：**
```markdown
# 面试评估报告

## 总体评分：{总分}/100

### 维度得分
| 维度 | 得分 | 权重 | 加权得分 |
|------|------|------|---------|
| 问题质量 | {得分}/40 | 40% | {加权} |
| 隐藏问题发现率 | {得分}/25 | 25% | {加权} |
| 控场能力 | {得分}/15 | 15% | {加权} |
| 合规性 | {得分}/10 | 10% | {加权} |
| 候选人体验 | {得分}/10 | 10% | {加权} |

## 详细分析

### ✅ 做得好的地方
1. {具体优点}
2. {具体优点}

### ⚠️ 需要改进的地方
1. {具体问题}
   - **问题：** {问题描述}
   - **建议：** {改进建议}

## 改进建议

### 短期改进（1 周内）
1. {改进建议}

### 长期改进（1 月内）
1. {改进建议}

## 下次练习建议
{下次练习建议}
```
```

**使用示例：**
```
Agent: "评估报告：
     总体评分：75/100
     - 问题质量：32/40（不错，但可以更深入）
     - 隐藏问题发现率：15/25（只发现了 1/3）
     - 控场能力：12/15（良好）
     - 合规性：10/10（完美）
     - 候选人体验：6/10（需改进）
     
     改进建议：
     1. 追问'你在团队中的角色是什么？'
     2. 提升亲和力，多微笑"
```

### Prompt 6：生成改进建议

**用途：** 根据评估结果，生成具体的改进建议

```markdown
你是一个面试培训专家。请根据评估结果，生成具体的改进建议。

**输入：**
- 评估结果：{评估报告}
- 薄弱环节：{薄弱环节列表}

**任务：**
1. 针对 3 个最薄弱的环节，生成具体改进建议
2. 改进建议必须具体、可执行、有示例
3. 区分"短期改进"（1 周内）和"长期改进"（1 月内）

**输出格式：**
```markdown
## 改进建议

### 短期改进（1 周内）

#### 1. {改进项}
**当前问题：** {问题描述}
**改进方法：** {具体方法}
**示例句式：**
- "{示例 1}"
- "{示例 2}"
**练习方法：** {如何练习}

### 长期改进（1 月内）

#### 1. {改进项}
**当前问题：** {问题描述}
**改进方法：** {具体方法}
**推荐场景：** {推荐练习的场景}
**目标：** {具体目标}
```
```

**使用示例：**
```
Agent: "改进建议：
     短期改进（1 周内）：
     1. 追问技巧
        - 背诵 3 个万能追问句式：
          - '具体是什么场景？'
          - '你的角色是什么？'
          - '你做了什么？'
     
     长期改进（1 月内）：
     1. 隐藏问题发现训练
        - 专门练习场景 2（有 3 个隐藏问题）
        - 目标：发现率从 33% 提升到 67%"
```

### Prompt 8：生成进步曲线

**用途：** 根据历史数据，生成进步曲线

```markdown
你是一个数据分析助手。请根据历史面试数据，生成进步曲线。

**输入：**
- 历史面试记录：{记录列表}

**任务：**
1. 统计每次面试的 5 维度得分
2. 生成进步曲线（可视化描述）
3. 分析进步趋势

**输出格式：**
```markdown
## 进步曲线

### 总分趋势
第 1 次：{总分}
第 2 次：{总分}（+{变化}）
第 3 次：{总分}（+{变化}）

### 维度进步

| 维度 | 第 1 次 | 第 2 次 | 第 3 次 | 趋势 |
|------|--------|--------|--------|------|
| 问题质量 | {得分} | {得分} | {得分} | ↗️ |
| 隐藏问题发现 | {得分} | {得分} | {得分} | ↗️ |
| 控场能力 | {得分} | {得分} | {得分} | → |

### 分析
{进步分析}
```
```

## 📊 Prompt 使用流程

### 完整面试流程

```
1. 用户: "开始面试模拟"
   → Agent 调用 Prompt 1（分析 JD 和简历）

2. 用户: [粘贴 JD 和简历]
   → Agent 调用 Prompt 2（生成 AI 候选人人设）
   → Agent 调用 Prompt 3（生成面试问题）

3. Agent: "准备好了吗？开始面试"
   → 用户: "开始"

4. [面试循环]
   → Agent 调用 Prompt 4（AI 候选人回答）
   → 用户: [追问]
   → Agent 调用 Prompt 4（AI 候选人回答）

5. [面试结束]
   → Agent 调用 Prompt 5（评估面试官表现）
   → Agent 调用 Prompt 6（生成改进建议）
   → Agent 调用 Prompt 8（生成进步曲线）
```

## 🎯 Prompt 最佳实践

### 对 Agent

1. **模块化调用：**
   - 每个 Prompt 只做一件事
   - 组合使用，完成复杂任务

2. **结构化输出：**
   - 使用 JSON、Markdown 等结构化格式
   - 便于解析和处理

3. **变量替换：**
   - 使用 `{变量名}` 占位符
   - 调用时替换为实际值

### 对用户

1. **明确输入：**
   - 提供 JD 和简历
   - 明确场景和目标

2. **及时反馈：**
   - 面试过程中及时追问
   - 面试结束后查看评估报告

3. **持续练习：**
   - 每周至少 2 次
   - 追踪进步曲线
