# 多 Agent 协作流程 - 面试题库准备

**目标：** 使用多个 Sub-Agent 协作，为 5 大行业准备高质量的面试题库

**时间：** 2026-03-11  
**负责人：** 虾球（主 Agent）协调

---

## 🎯 总体策略

### 核心思路

**不是"从零创建"，而是"收集 + 优化"：**
1. 先搜索和收集现有的开源题库
2. 用 Agent 过滤和清洗
3. 用 Agent 针对我们 17 个场景定制化
4. 用 Agent 验证质量

### 为什么多 Agent？

| 单 Agent | 多 Agent |
|---------|---------|
| 顺序执行，时间长 | 并行执行，时间短 |
| 容易遗漏 | 互相验证，减少遗漏 |
| 单一视角 | 多个专家视角 |

## 👥 Agent 角色分配

### 主 Agent（Coordinator）：虾球

**职责：**
- 协调 5 个行业 Agent
- 汇总结果
- 质量把控
- 最终决策

### Sub-Agent 1：资源猎手（Resource Hunter）

**任务：** 搜索和收集开源面试题库

**搜索策略：**
```
1. GitHub 搜索
   - 关键词："interview questions"
   - 语言：中文、英文
   - 排序：stars 最多

2. Google 搜索
   - 关键词："开源面试题库"、"面试问题集"
   - 文件类型：JSON、Markdown

3. 专业网站
   - 掘金、知乎、CSDN
   - 牛客网、力扣
```

**输出格式：**
```json
{
  "industry": "互联网/科技",
  "resources": [
    {
      "source": "GitHub",
      "url": "https://github.com/xxx/interview-questions",
      "stars": 1000,
      "description": "包含 500+ 面试题",
      "license": "MIT"
    }
  ]
}
```

---

### Sub-Agent 2：题库清洗者（Question Cleaner）

**任务：** 清洗和标准化收集的题库

**清洗规则：**
```
1. 去重
   - 问题内容相似度 > 80% → 去重

2. 分类
   - 按岗位分类（产品经理、工程师、运营）
   - 按难度分类（初级、中级、高级）

3. 标准化
   - 统一格式（JSON）
   - 统一字段（问题、答案、标签）
```

**输出格式：**
```json
{
  "industry": "互联网/科技",
  "role": "产品经理",
  "questions": [
    {
      "id": "q001",
      "question": "请介绍一个你负责的产品项目",
      "category": "项目经验",
      "difficulty": "中级",
      "tags": ["STAR", "追问"]
    }
  ]
}
```

---

### Sub-Agent 3：场景定制者（Scenario Customizer）

**任务：** 根据我们的 17 个场景，定制化问题

**定制规则：**
```
1. 对接场景
   - 场景 1：产品经理初试
   - 场景 2：产品经理复试
   - ...

2. 隐藏问题匹配
   - 为每个问题标注"可触发隐藏问题"
   - 例如："你在团队中的角色是什么？" → 可触发"过度包装"

3. STAR 法则优化
   - 为每个问题添加"追问示例"
   - 例如："具体是什么场景？你的任务是什么？"
```

**输出格式：**
```json
{
  "scenario_id": "scenario_001",
  "scenario_name": "产品经理初试",
  "questions": [
    {
      "question": "请介绍一个你负责的产品项目",
      "follow_up": [
        "具体是什么场景？",
        "你的角色是什么？",
        "你做了什么？",
        "结果如何？"
      ],
      "triggers_hidden_issue": true,
      "hidden_issue_type": "过度包装"
    }
  ]
}
```

---

### Sub-Agent 4：质量验证者（Quality Validator）

**任务：** 验证问题的质量和合理性

**验证规则：**
```
1. 相关性验证
   - 问题是否和岗位 JD 相关？
   - 问题是否和场景目标匹配？

2. 合规性验证
   - 问题是否涉及隐私？（婚姻、政治、宗教）
   - 如果是，标记为"高风险"

3. 专业性验证
   - 问题是否专业？
   - 是否符合行业标准？

4. 实用性验证
   - 问题是否真实？
   - 是否有实际价值？
```

**输出格式：**
```json
{
  "validation_report": {
    "total_questions": 100,
    "passed": 85,
    "failed": 15,
    "issues": [
      {
        "question_id": "q015",
        "issue": "涉及隐私问题",
        "severity": "高风险",
        "suggestion": "删除或修改"
      }
    ]
  }
}
```

## 🔄 工作流程

### Phase 1：资源收集（并行）

```
┌─────────────────────────────────────────────┐
│  主 Agent（虾球）                            │
│  任务：分配 5 个行业给 5 个 Sub-Agent        │
└────────────┬────────────────────────────────┘
             │
     ┌───────┴───────┬───────┬───────┬───────┐
     │               │       │       │       │
     ▼               ▼       ▼       ▼       ▼
 [Sub-Agent 1]   [Sub-Agent 2] ... (5 个)
 互联网/科技      金融/FinTech   (并行执行)
```

**每个 Sub-Agent 的任务：**
1. 搜索行业面试题库（GitHub、Google、专业网站）
2. 收集至少 10 个资源
3. 返回资源列表（URL、stars、描述）

**时间：** 10 分钟（并行）

---

### Phase 2：题库清洗（并行）

```
┌─────────────────────────────────────────────┐
│  Sub-Agent 2（题库清洗者）                   │
│  任务：清洗每个行业的题库                     │
└────────────┬────────────────────────────────┘
             │
     ┌───────┴───────┬───────┬───────┬───────┐
     │               │       │       │       │
     ▼               ▼       ▼       ▼       ▼
 [互联网题库]   [金融题库] ... (5 个)
 去重、分类、标准化   (并行执行)
```

**每个 Sub-Agent 的任务：**
1. 去重（问题相似度 > 80%）
2. 分类（岗位、难度）
3. 标准化（JSON 格式）
4. 返回清洗后的题库

**时间：** 15 分钟（并行）

---

### Phase 3：场景定制（并行）

```
┌─────────────────────────────────────────────┐
│  Sub-Agent 3（场景定制者）                   │
│  任务：根据 17 个场景定制问题                 │
└────────────┬────────────────────────────────┘
             │
     ┌───────┴───────┬───────┬───────┬───────┐
     │               │       │       │       │
     ▼               ▼       ▼       ▼       ▼
 [场景 1-3]     [场景 4-6] ... (5 组)
 互联网场景     金融场景    (并行执行)
```

**每个 Sub-Agent 的任务：**
1. 对接 17 个场景
2. 为每个问题添加"隐藏问题触发"
3. 添加"追问示例"
4. 返回定制化题库

**时间：** 20 分钟（并行）

---

### Phase 4：质量验证（串行）

```
┌─────────────────────────────────────────────┐
│  Sub-Agent 4（质量验证者）                   │
│  任务：验证所有问题的质量                     │
└────────────┬────────────────────────────────┘
             │
             ▼
        [验证所有题库]
        相关性、合规性、专业性、实用性
             │
             ▼
        [验证报告]
        通过：85 / 失败：15
```

**Sub-Agent 4 的任务：**
1. 验证相关性（是否和 JD 相关）
2. 验证合规性（是否涉及隐私）
3. 验证专业性（是否符合行业标准）
4. 验证实用性（是否有实际价值）
5. 返回验证报告

**时间：** 15 分钟（串行）

---

### Phase 5：JSON 格式化（并行）

```
┌─────────────────────────────────────────────┐
│  Sub-Agent 5（JSON 格式化者）                │
│  任务：格式化为最终的 JSON 文件               │
└────────────┬────────────────────────────────┘
             │
     ┌───────┴───────┬───────┬───────┬───────┐
     │               │       │       │       │
     ▼               ▼       ▼       ▼       ▼
 [tech-pm.json] [fintech-analyst.json] ...
```

**每个 Sub-Agent 的任务：**
1. 格式化为标准 JSON
2. 保存到 `assets/questions/`
3. 返回文件路径

**时间：** 10 分钟（并行）

---

### Phase 6：汇总和决策（主 Agent）

```
┌─────────────────────────────────────────────┐
│  主 Agent（虾球）                            │
│  任务：汇总结果，质量把控，最终决策            │
└─────────────────────────────────────────────┘

输入：
- 5 个行业的题库（JSON）
- 质量验证报告

任务：
1. 检查每个题库的质量
2. 验证问题数量（每个场景 5-12 个）
3. 最终决策：通过 / 打回 / 修改

输出：
- 17 个场景的面试题库（JSON 文件）
- 最终报告
```

**时间：** 10 分钟

## 🚀 执行计划

为每个 Sub-Agent 准备详细的任务描述和示例。

### Step 2：并行启动 Sub-Agent

### Step 3：监控和协调

主 Agent 监控每个 Sub-Agent 的进度，及时协调问题。

收集所有 Sub-Agent 的结果，质量把控，最终决策。
