# 简历匹配和面试提纲 - 使用示例

**目的：** 展示在用户发送 JD 和简历后，Agent 如何生成简历匹配分析和面试提纲

---

## 📋 输入示例

### 岗位 JD

```
【产品经理（3-5 年经验）】

岗位职责：
- 负责产品规划和需求分析
- 协调设计和开发团队
- 数据驱动决策
- 跨部门协作

任职要求：
- 3-5 年产品经理经验
- 有 B 端产品经验优先
- 熟悉数据分析方法
- 优秀的沟通能力
- 有金融行业经验者优先
```

### 候选人简历

```
【张三】
男，28 岁，3 年产品经理经验

工作经历：
2021-2024：某电商公司，产品经理
- 负责 B2C 电商产品的规划和迭代
- 协调开发和设计团队完成产品上线
- 通过数据驱动优化用户体验

2019-2021：某金融科技公司，产品专员
- 协助产品经理完成需求文档
- 参与用户调研和数据分析

教育背景：
- 华中科技大学，信息管理 2018 届
```

## 🎯 Agent 输出示例

### Step 1：简历匹配分析

```
【简历匹配分析】

匹配度：70%

✅ 岗位要求：3-5 年产品经理经验 ✅
✅ 核心技能：需求分析、数据分析 ✅
✅ 沟通能力：优秀的沟通能力 ✅

⚠️ 经验略低：3 年经验（要求 3-5 年，刚达标）
⚠️ 行业不匹配：电商 → 金融（岗位要求有金融优先，但候选人是电商）
⚠️ 缺少 B 端经验：只有 C 端经验（岗位明确要求 B 端优先）
⚠️ 数据分析能力未知：简历只提到"用数据驱动"，未展示具体案例

💡 面试建议：
1. 重点考察 B 端行业知识（电商 → 金融）
2. 深入追问数据分析案例（具体方法、工具、结果）
3. 评估跨行业适应能力
```

### Step 2：面试提纲（5 个问题）

```
【面试提纲】

问题 1：请介绍一个你负责的电商产品项目
- 目的：考察项目经验和表达能力
- 隐藏问题：过度包装项目参与度
- 追问技巧：
  - "具体是什么产品？用户规模？"
  - "你在团队中的角色是什么？"
  - "你具体负责哪些功能？"
  - "遇到了什么挑战？"

问题 2：你怎么理解 B 端产品？和 C 端有什么不同？
- 目的：考察 B 端行业知识
- 隐藏问题：行业认知浅
- 追问技巧：
  - "你认为 B 端用户和 C 端用户有什么不同？"
  - "B 端产品的关键成功因素是什么？"
  - "你觉得金融产品的 B 端用户需要什么？"

问题 3：你用过哪些数据分析方法？
- 目的：考察数据分析能力
- 隐藏问题：技术深度不足
- 追问技巧：
  - "你用什么工具？"
  - "具体分析什么数据？"
  - "怎么支撑产品决策？"
  - "有没有数据驱动的具体案例？"

问题 4：你为什么要从电商转到金融行业？
- 目的：考察职业规划动机
- 隐藏问题：离职真实原因
- 追问技巧：
  - "具体是什么原因？"
  - "金融产品的哪一点吸引你？"
  - "你为转行做了哪些准备？"
  - "你怎么证明你的决心？"

问题 5：你遇到过最大的产品挑战是什么？怎么解决的？
- 目的：考察问题解决能力和抗压能力
- 隐藏问题：夸大个人贡献
- 追问技巧：
  - "具体是什么挑战？"
  - "你的角色是什么？"
  - "你采取了什么行动？"
  - "结果如何？量化指标？"
```

### Step 3：场景确认

```
✅ 准备就绪，开始面试？

你可以：
1. 直接说"开始"
2. 调整面试提纲（增删问题）
3. 修改场景（切换到「产品经理复试」）
```

## 🔍 隐藏问题触发逻辑

### 问题 1：请介绍一个你负责的电商产品项目

**AI 候选人表面回答：**
```
我负责过一个社交电商产品，帮助用户发现兴趣社群。
用户留存提升了 30%，DAU 达到 100 万。
```

**追问：**
```
用户: "你在团队中的角色是什么？具体负责哪部分？"

AI 候选人 [停顿 1 秒]:
"呃...主要是配合产品经理做一些辅助工作，
核心功能是另一个同事主导的。
具体数据记不清了，主要是通过自然增长。"
```

**信号识别：**
- ⚠️ 停顿 1 秒
- ⚠️ 模糊回答（"具体数据记不清了"）
- ✅ 触发隐藏问题：过度包装项目参与度

---

### 问题 4：你为什么要从电商转到金融行业？

**AI 候选人表面回答：**
```
我希望接触更多元化的产品，金融产品更有挑战性。
而且金融行业发展前景更好。
```

**追问：**
```
User: "具体是什么原因？"
AI 候选人: "主要是电商行业竞争太激烈了，
我觉得金融行业机会更多。"
```

**信号识别：**
- ⚠️ 只谈行业趋势，不谈个人动机
- ⚠️ 没有具体转行准备
- ⚠️ 可能是被迫离职（电商行业裁员？）
- ✅ 触发隐藏问题：离职真实原因未明

**更深入追问：**
```
User: "你在上家公司的绩效怎么样？
AI 候选人: [转移话题]
```

## 📊 评估报告示例

```
【面试评估报告】

总分：72/100

### 维度得分
| 维度 | 得分 | 权重 | 加权得分 |
|------|------|------|---------|
| 问题质量 | 30/40 | 40% | 30 |
| 隐藏问题发现率 | 15/25 | 25% | 15 |
| 控场能力 | 12/15 | 15% | 12 |
| 合规性 | 10/10 | 10% | 10 |
| 候选人体验 | 5/10 | 10% | 5 |

### ✅ 做得好的地方

1. **问题相关性高** - 所有问题和 JD 匹配
2. **使用了追问技巧** - 追问了具体角色
3. **完全合规** - 没有敏感问题

### ⚠️ 需要改进的地方

1. **隐藏问题发现率低（15/25）**
   - **问题：** 只发现了 1/2 个隐藏问题
   - **建议：** 深入追问数据分析案例（具体方法、工具、结果）

2. **候选人体验一般（5/10）**
   - **问题：** 亲和力不足
   - **建议：** 多微笑，让候选人更舒适

### 🔧 改进建议

#### 短期改进（1 周内）

1. **练习追问技巧**
   - 背诵 3 个万能追问句式：
     - "具体是什么场景？"
     - "你的角色是什么？"
     - "你做了什么？"

2. **提升亲和力**
   - 面试前微笑
   - 候选人回答时点头
   - 让候选人更舒适

#### 长期改进（1 月内）

1. **数据分析能力训练**
   - 学习常见数据分析工具（SQL、Python、Tableau）
   - 实际项目经验

2. **跨行业知识补充**
   - 了解金融行业基础知识
   - 研究 B 端产品案例

### 📈 进步曲线

这是你第 1 次练习，继续加油！

---
```

## 🎯 核心价值

### 对面试官

**面试前：**
- 提前了解候选人优势和风险
- 准备针对性的问题
- 避免"盲试"

**面试中：**
- 实时识别隐藏问题信号
- 优化追问策略
- 保持专业和亲和力

**面试后：**
- 量化评估自己的表现
- 获得具体改进建议
- 追踪进步曲线

### 对企业

**招聘前：**
- 快速筛选候选人
- 识别高风险候选人
- 降低招聘失误成本

**招聘后：**
- 标准化面试流程
- 提升面试官能力
- 积累招聘数据

## 🚀 使用建议

### 什么时候用？

1. **新面试官入职培训**
   - 让新面试官快速上手
   - 降低试错成本

2. **面试前热身**
   - 重要面试前 15 分钟
   - 保持状态

3. **团队能力提升**
   - 定期练习，提升能力
   - 标准化面试流程

### 怎么用？

1. **直接使用：**
   ```
   "开始面试模拟"
   [粘贴 JD 和简历]
   ```

2. **自定义场景：**
   ```
   "我要练习「金融分析师」面试，难度 L3"
   ```

3. **对比分析：**
   ```
   "对比我和行业平均水平的差异"
   ```
