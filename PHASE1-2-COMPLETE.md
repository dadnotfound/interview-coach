1. **提交新问题** - 通过 Fork + PR

**社区共建激励：**
- 观察者（0 次贡献）

## 💡 关键决策

### 决策 1：开源共建策略

**选择：** 将题库设计为可扩展、标准化的格式，支持社区共建

**原因：**
1. 单人无法覆盖所有行业和岗位
2. 社区共建可以快速扩充题库
3. 标准化格式便于自动化处理

**影响：**
- 需要设计清晰的贡献指南
- 需要建立审核机制
- 需要创建 GitHub 模板

### 决策 2：多 Agent 并行执行

**选择：** 使用 5 个 Sub-Agent 并行执行资源收集和题库清洗

**原因：**
1. 时间效率：并行 80 分钟 vs 串行 200 分钟
2. 质量保证：互相验证，减少遗漏
3. 专业性：每个 Agent 专注一个行业

**影响：**
- 速率限制问题（1 个 Agent 失败）
- 需要手动补充 AI 行业题库
